DevOps & Automation

CI/CD pipelines, Docker images, website publishing, automated snapshots, and release management

Delivery Pipeline

A reliable, repeatable release process ensures DeadLink stays stable and easy to deploy

⚙️

CI Checks

Jest tests and smoke tests on every push and PR. Optional linting. Fast feedback loops keep quality high.

Workflow: .github/workflows/ci.yml

🏷️

Releases

Tagging vX.Y.Z creates a GitHub Release with automated notes. Publish stable, traceable builds quickly.

Workflow: .github/workflows/release.yml

🐳

Docker Images

Multi-arch (amd64/arm64) builds pushed to GHCR. Use version tags or latest for deployments.

Workflow: .github/workflows/docker.yml → ghcr.io/thecdrz/deadlink

📰

Website Publishing

GitHub Pages builds the docs/ site on every change. Keep docs fresh without manual steps.

Workflow: .github/workflows/pages.yml

📸

UI Snapshots

Puppeteer renders sample embeds and saves PNGs to screenshots/ and docs/assets/snapshots/.

Script: scripts/snapshot-embeds.js — Run: npm run snapshots

🚀

Docker Run

Run DeadLink with env vars (Discord, telnet). DEV_MODE=true enables simulated telnet for demos.

Docs: See README → Docker
